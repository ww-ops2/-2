# 采购单增强功能说明

## 📋 功能概述

本次更新为采购单模块增加了以下核心功能，大幅提升采购录入效率和数据规范性：

### ✨ 主要改进

1. **多物品采购支持** - 通过"+"按钮可一次性添加多个不同类别的物品
2. **智能类别管理** - 支持下拉选择、直接输入新增、自动同步到库存分类
3. **品牌型号历史记录** - 自动保存并智能提示，避免重复输入
4. **唯一商品编码** - 自动生成SKU编码，确保每个商品的唯一性
5. **优化的用户体验** - 精美的表格样式、流畅的交互动画

---

## 🎯 详细功能说明

### 1️⃣ 多物品采购（"+"按钮）

**功能描述：**
- 在采购明细表格中，点击"+ 添加物品"按钮可动态添加新行
- 每次点击都会生成一行新的物品录入区域
- 支持一次采购多种不同类别的商品

**使用方法：**
```
1. 打开"新建采购单"模态框
2. 点击表格底部的"+ 添加物品"按钮
3. 填写该行物品的详细信息
4. 重复步骤2-3添加更多物品
5. 点击"提交采购单"完成创建
```

**界面效果：**
- 每行包含：类别、物品名称、商品编码、品牌、型号、数量、单位、单价、金额
- 删除按钮（✕）可快速移除不需要的行
- 悬停高亮效果，清晰标识当前操作行

---

### 2️⃣ 智能类别管理

**功能特性：**

#### A. 下拉选择 + 直接输入
- 类别字段使用 `datalist` 控件，兼具下拉选择和自由输入的优势
- 可从现有类别中选择，也可直接输入新类别名称
- 输入时会自动匹配已有类别（模糊搜索）

#### B. 自动创建新类别
- 当输入不存在的类别名称时，系统会自动创建该类别
- 自动生成唯一的类别编码（格式：CAT001, CAT002...）
- 自动同步到库存分类系统，保持数据一致性

#### C. 手动新增类别
- 点击类别旁边的"+"按钮，打开类别新增对话框
- 可填写类别名称和备注信息
- 适合批量预先定义类别的场景

**类别数据结构：**
```javascript
{
  code: "CAT001",           // 类别编码
  name: "饮品",             // 类别名称
  remark: "",               // 备注说明
  created_at: "2026-06-09T..."  // 创建时间
}
```

**使用示例：**
```
场景1：选择已有类别
- 点击类别输入框
- 从下拉列表中选择"饮品"、"食品"等现有类别

场景2：输入新类别
- 直接在类别输入框中输入"电子产品"
- 失去焦点后，系统自动创建该类别
- 下次录入时即可在下拉列表中看到"电子产品"

场景3：手动新增
- 点击类别旁的"+"按钮
- 在弹窗中填写类别信息
- 点击"保存"完成创建
```

### 3️⃣ 品牌/型号历史记录

**功能描述：**
- 系统自动记录每个物品名称对应的品牌和型号历史
- 再次输入相同物品名称时，会自动显示历史品牌/型号供选择
- 基于 HTML5 `datalist` 实现智能提示

**工作原理：**
```javascript
// 数据存储结构
brandHistory = {
  "矿泉水": ["农夫山泉", "怡宝", "百岁山"],
  "方便面": ["康师傅", "统一", "今麦郎"]
}

modelHistory = {
  "矿泉水": ["550ml", "1L", "5L"],
  "方便面": ["桶装", "袋装", "盒装"]
}
```

**使用方法：**
```
1. 第一次录入：
   - 物品名称：矿泉水
   - 品牌：农夫山泉（手动输入）
   - 型号：550ml（手动输入）
   - 系统自动保存这些历史记录

2. 第二次录入：
   - 物品名称：矿泉水
   - 点击品牌输入框，下拉显示：农夫山泉、怡宝、百岁山
   - 点击型号输入框，下拉显示：550ml、1L、5L
   - 可直接选择或继续输入新值
```

**优势：**
- ✅ 减少重复输入，提高录入效率
- ✅ 保证数据一致性，避免拼写错误
- ✅ 智能学习用户的录入习惯
- ✅ 支持多选，灵活性强

### 4️⃣ 唯一商品编码生成

**编码规则：**
- 格式：`SKU` + 5位数字（如：SKU00001, SKU00002...）
- 自动递增，确保全局唯一性
- 持久化存储，刷新页面后继续递增

**技术实现：**
```javascript
// 计数器存储在 localStorage
let counter = parseInt(localStorage.getItem('itemCodeCounter') || '1');
const code = `SKU${String(counter).padStart(5, '0')}`;
counter++;
localStorage.setItem('itemCodeCounter', counter.toString());
```

**特点：**
- 🔒 **唯一性保证**：基于持久化计数器，即使跨会话也保持唯一
- 📊 **可读性强**：SKU前缀清晰标识商品编码
- 🔄 **自动管理**：无需人工干预，系统自动生成
- 💾 **数据安全**：本地存储，不受网络影响

**显示效果：**
- 商品编码字段为只读状态
- 灰色背景突出显示，便于识别
- 等宽字体（monospace），对齐美观

## 🔧 技术实现要点

### 1. 类别管理
- 使用 `<datalist>` 实现下拉+输入双模式
- 失焦事件（blur）检测新类别并自动创建
- localStorage 持久化存储类别数据
- 实时刷新所有类别下拉列表

### 2. 品牌型号历史
- 以物品名称为键，存储品牌/型号数组
- 输入物品名称后，动态更新对应 datalist
- 失焦时保存新输入的历史记录
- 去重处理，避免重复数据

### 3. 商品编码
- localStorage 存储全局计数器
- 每次生成后递增并保存
- 格式化输出：SKU + 5位数字（前导零）
- 页面加载时恢复计数器状态

### 4. 样式优化
- CSS transition 实现平滑过渡
- @keyframes 实现入场动画
- :hover 伪类实现交互反馈
- :focus 伪类实现焦点高亮

## 💡 最佳实践建议

### 1. 类别命名规范
- ✅ 推荐：简洁明确（如"文具"、"饮品"、"电子设备"）
- ❌ 避免：过于具体（如"黑色中性笔专用类别"）
- 💡 原则：类别应该是物品的分组，而非单个物品

### 2. 品牌型号录入
- 首次录入时尽量完整准确
- 同一物品保持品牌型号的一致性
- 利用历史记录快速选择，减少输入错误

### 3. 商品编码使用
- SKU编码仅供系统内部使用，无需记忆
- 可通过物品名称、类别进行搜索
- 导出报表时会显示SKU编码便于追踪

### 4. 多物品采购技巧
- 先添加所有物品行，再统一填写详细信息
- 相似物品可以连续添加，利用历史记录加速
- 定期清理不再使用的类别，保持数据整洁

## 🐛 常见问题

### Q1: 为什么我输入的类别没有出现在下拉列表中？
**A:** 请确保：
1. 输入完成后点击其他地方（失焦）触发保存
2. 刷新页面后类别会永久保存
3. 检查浏览器是否禁用了 localStorage

### Q2: 品牌型号历史记录如何清除？
**A:** 目前暂不支持手动清除，历史记录会永久保存。如需清理，可在浏览器控制台执行：
```javascript
localStorage.removeItem('brandHistory');
localStorage.removeItem('modelHistory');
```

### Q3: 商品编码可以自定义吗？
**A:** 当前版本采用自动生成的SKU编码，暂不支持自定义。这样可以确保编码的唯一性和规范性。

### Q4: 如果误删了一行，能恢复吗？
**A:** 提交前可以重新添加。提交后无法单独恢复某一行，但可以查看采购单详情查看所有物品信息。

### Q5: 类别同步到库存分类有什么作用？
**A:** 确保采购系统和库存系统的分类保持一致，便于后续：
- 库存统计按分类汇总
- 报表分析更加准确
- 数据管理规范统一
