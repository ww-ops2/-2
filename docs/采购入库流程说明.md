# 采购入库流程使用说明

## 📋 核心设计原则

**所有库存明细的新增来源必须来自采购入库**，确保数据来源可追溯。

- ✅ **采购员**：创建采购单 → 提交至仓库待入库
- ✅ **仓库管理员**：确认入库 → 自动生成库存明细
- ✅ **管理员**：可以添加历史数据（特殊权限）
- ❌ **其他角色**：不能直接创建库存物品

---

## 🔄 完整业务流程

```
采购单创建 → 提交流程 → 仓库待入库 → 仓库确认 → 生成库存明细
   ↓                                              ↓
 Excel导入/手动录入                              库存系统自动更新
```

---

## 🛒 一、采购单管理

### 1.1 新建采购单（手动录入）

**操作步骤：**
1. 点击"采购单管理"菜单
2. 点击"+ 新建采购单"按钮
3. 填写采购信息：
   - 采购日期（默认今天）
   - 采购人姓名
   - 供应商（可选）
4. 添加采购明细：
   - 点击"+ 添加物品"按钮
   - 填写：物品名称*、品牌、型号、数量*、单位、单价
   - 金额自动计算（数量 × 单价）
   - 可添加多个物品
5. 填写备注（可选）
6. 点击"提交采购单"

**系统行为：**
- 自动生成采购单号（格式：PO20260609001）
- 状态设置为"待入库"
- 流转至仓库管理模块

---

### 1.2 Excel导入采购单

**操作步骤：**

#### 步骤1：下载模板
- 点击"📄 导入Excel"按钮
- 点击"📥 下载Excel模板"
- 模板包含示例数据，按照格式填写

#### 步骤2：填写数据
在Excel中填写以下列：
| 列名 | 说明 | 必填 |
|------|------|------|
| 采购日期 | 格式：2026-06-09 | ✅ |
| 采购人 | 采购人姓名 | ✅ |
| 供应商 | 供应商名称 | ❌ |
| 物品名称 | 物品名称 | ✅ |
| 品牌 | 品牌信息 | ❌ |
| 型号 | 型号规格 | ❌ |
| 数量 | 采购数量（>0） | ✅ |
| 单位 | 计量单位 | ❌ |
| 单价 | 单价（元） | ❌ |
| 备注 | 备注说明 | ❌ |

**提示：**
- 同一采购日期和采购人会自动合并为一个采购单
- 不同日期或采购人会生成多个采购单

#### 步骤3：上传预览
- 点击上传区域或拖拽文件
- 系统自动解析Excel
- 显示数据预览表格
- 显示错误信息（如有）
- 显示统计：记录数、总金额

#### 步骤4：确认导入
- 检查预览数据无误
- 点击"确认导入"
- 系统自动创建采购单

**支持格式：**
- `.xlsx` (Excel 2007+)
- `.xls` (Excel 97-2003)

---

### 1.3 查看采购单

**操作：**
- 在采购单列表中点击"查看"按钮
- 显示详细信息：
  - 采购单号、日期、采购人、供应商
  - 总金额、状态
  - 所有物品明细（名称、品牌、型号、数量、单价、金额）

## 📥 二、入库管理

### 2.1 待入库列表

**查看位置：**
- 采购单管理页面：状态为"待入库"的采购单有"入库"按钮
- 入库管理页面：查看所有入库记录

**权限控制：**
- 只有**仓库管理员**角色可以确认入库
- 其他角色只能查看

---

### 2.2 确认入库

**操作步骤：**
1. 在采购单列表中找到"待入库"状态的采购单
2. 点击"入库"按钮（仅仓库管理员可见）
3. 系统打开入库确认模态框，显示：
   - 采购单信息（单号、日期、采购人、供应商）
   - 入库日期（默认今天）
   - 批次号（自动生成，可修改）
   - 入库明细表格

4. **核对入库明细：**
   - 查看每个物品的采购数量
   - 修改"实收数量"（如有差异）
   - 系统自动计算金额

5. 填写验收备注（如有差异必须说明）

6. 点击"确认入库"

**系统行为：**
- 创建入库记录（单号格式：SI + 时间戳）
- 更新采购单状态为"已入库"
- **自动生成库存明细**：
  - 如果物品已存在：增加库存数量
  - 如果物品不存在：创建新库存记录
  - 标记来源为"purchase"（采购）
  - 记录最后入库日期和批次号

---

### 2.3 查看入库记录

**操作：**
- 进入"入库管理"菜单
- 查看所有入库记录
- 可按状态筛选（全部/待确认/已完成）
- 点击"查看详情"查看完整信息

**显示信息：**
- 入库单号、关联采购单号
- 入库日期、批次号
- 物品种类数、总数量
- 确认人、确认时间
- 详细明细（采购数量 vs 实收数量）

## 📦 三、库存明细生成规则

### 3.1 自动生成逻辑

**触发条件：**
- 仓库管理员确认入库后

**生成规则：**

1. **物品已存在**（名称+品牌+型号匹配）：
   ```javascript
   existingItem.stock += actual_quantity
   existingItem.last_stockin_date = stockin_date
   existingItem.last_stockin_batch = batch_code
   ```

2. **物品不存在**：
   ```javascript
   {
     id: 自动生成,
     code: "ITEM001",  // 自动编号
     name: 物品名称,
     brand: 品牌,
     model: 型号,
     category: "未分类",  // 需后续分类
     stock: 实收数量,
     unit: 单位,
     safety_stock: 10,  // 默认安全库存
     last_stockin_date: 入库日期,
     last_stockin_batch: 批次号,
     source: "purchase",  // 标记来源
     created_at: 当前时间
   }
   ```

---

### 3.2 数据来源追溯

**所有库存物品的来源标记：**
- `source: "purchase"` - 来自采购入库
- `source: "history"` - 管理员添加的历史数据

**查询时可追溯：**
- 通过 `last_stockin_batch` 找到对应批次
- 通过批次号找到入库记录
- 通过入库记录找到采购单
- 形成完整的溯源链条

## 👑 四、管理员特殊权限

### 4.1 添加历史数据

**仅限管理员角色：**
1. 进入"库存概览"页面
2. 点击"+ 新增物品"按钮
3. 填写物品信息
4. 设置初始库存
5. 保存后标记为 `source: "history"`

**注意：**
- 此功能仅用于系统初始化或数据迁移
- 正常业务流程应通过采购入库
- 所有历史数据都有明确标记

## 💡 六、使用技巧

### 6.1 批量导入最佳实践

1. **按供应商分组**：同一供应商的物品放在一个Excel文件
2. **按日期分组**：同一天采购的物品放在一起
3. **数据校验**：导入前仔细检查数量和单价
4. **保留备份**：导入前备份原始Excel文件

### 6.2 入库验收注意事项

1. **实物核对**：逐一核对物品名称、品牌、型号
2. **数量验收**：如实填写实收数量
3. **质量检查**：在备注中记录质量问题
4. **批次管理**：批次号可用于追溯和质量追踪

### 6.3 数据一致性保障

- ✅ 采购单一旦提交不可修改（如需修改请取消后重新创建）
- ✅ 入库确认后自动生成库存，不可回退
- ✅ 所有操作都有日志记录
- ✅ 定期导出数据备份

## 🎯 八、常见问题

### Q1: 导入Excel时提示"数据列数不足"？
**A:** 检查Excel文件格式，确保至少有6列数据（采购日期、采购人、物品名称、数量等必填字段）

### Q2: 为什么看不到"入库"按钮？
**A:** 只有仓库管理员角色才能看到入库按钮，请确认登录账号的角色

### Q3: 入库后发现数量填错了怎么办？
**A:** 目前版本入库后不可修改，建议联系管理员处理。后续版本会添加入库调整功能

### Q4: 如何区分不同批次的同一物品？
**A:** 通过 `last_stockin_batch` 字段可以追溯到最后一次入库的批次号

### Q5: 库存数量为什么不准确？
**A:** 检查是否有未确认的入库单，库存只在入库确认后才会更新
