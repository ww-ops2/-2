##  核心改进

### 1️ **供应商分组架构** ✅

#### 之前的问题
```
❌ 一个采购单只能有一个供应商
❌ 所有商品都归属同一个供应商（不合理）
 无法区分不同供应商的商品
```

#### 新的架构
```
✅ 支持多个供应商，每个供应商独立卡片
✅ 每个供应商有自己的商品列表
✅ 可以添加/删除任意供应商分组
✅ 每个分组有独立的小计金额
```

#### 视觉效果
```
┌─────────────────────────────────────┐
│ 供应商A: XX贸易公司         [删除]   │
├─────────────────────────────────────┤
│ SKU000001                           │

│ [数量] [单位] [单价] [金额] [✕][+]  │

│ 小计: 150.00                        │

┌─────────────────────────────────────┐
│ 供应商B: YY办公用品店       [删除]   │

总计: 230.00
```

### 2️⃣ **类别编码前缀系统** ✅

#### 之前的逻辑
```
全局计数器: SKU00001, SKU00002, SKU00003...
问题: 所有类别混在一起，无法区分
```

#### 新的逻辑
```
按类别独立计数:
├─ 饮品 (SKU) → SKU000001, SKU000002, SKU000003...
├─ 食品 (SKP) → SKP000001, SKP000002, SKP000003...
└─ 日用品 (SKD) → SKD000001, SKD000002, SKD000003...

优势:
✅ 一眼看出商品类别
✅ 各类别独立管理
✅ 便于统计和查询
```

#### 自动识别规则
```javascript
if (类别名包含'饮品')    → 前缀: SKU
if (类别名包含'食品')    → 前缀: SKP  
if (类别名包含'日用')    → 前缀: SKD
else                     → 前缀: SK + 序号
```

#### 3. 按类别生成编码
```javascript
function generateItemCodeByCategory(categoryCode) {
  // 获取该类别的计数器
  let counter = categoryCodeCounters[categoryCode] || 1;
  
  // 生成编码: 前缀 + 6位数字
  const code = `${categoryCode}${String(counter).padStart(6, '0')}`;
  
  // 更新计数器并保存
  categoryCodeCounters[categoryCode]++;
  localStorage.setItem(`categoryCounter_${categoryCode}`, counter);
  
  return code;
}
```

#### 4. 自动创建类别
```javascript
function autoCreateCategory(categoryName) {
  // 根据类别名智能分配前缀
  let codePrefix;
  if (categoryName.includes('饮品')) codePrefix = 'SKU';
  else if (categoryName.includes('食品')) codePrefix = 'SKP';
  else if (categoryName.includes('日用')) codePrefix = 'SKD';
  else codePrefix = 'SK' + String(categories.length + 1).padStart(2, '0');
  
  categories.push({ code: codePrefix, name: categoryName });
  return categories[categories.length - 1];
}
```

#### 5. 更新小计和总计
```javascript
// 更新单个分组小计
function updateGroupSubtotal(row) {
  const group = row.closest('.supplier-group');
  let subtotal = 0;
  // 累加该分组所有行的金额
  group.querySelectorAll('.row-amount').forEach(el => {
    subtotal += parseFloat(el.textContent) || 0;
  });
  group.querySelector('.group-subtotal').textContent = subtotal.toFixed(2);
}

// 更新总金额
function updatePurchaseTotal() {
  let total = 0;
  // 累加所有分组的小计
  document.querySelectorAll('.group-subtotal').forEach(el => {
    total += parseFloat(el.textContent) || 0;
  });
  document.getElementById('purchase-total-amount').textContent = total.toFixed(2);
}
```

## 💡 使用小贴士

### 高效录入技巧
1. **批量添加供应商**: 先点几次"+ 添加供应商"，再逐个填写
2. **同类商品集中**: 将同一供应商的商品连续添加，减少切换
3. **利用历史记录**: 品牌、型号会自动保存，下次直接选择
4. **关注小计**: 每个供应商的小计帮助核对金额

### 最佳实践
- 供应商名称尽量完整（如"XX贸易有限公司"而非"XX"）
- 类别命名规范（如"饮品"而非"饮料类"），便于自动识别前缀
- 定期清理无用供应商分组，保持界面整洁
- 提交前检查各分组小计是否与预期一致
