### 3. 唯一编码生成算法

**持久化计数器：**
```javascript
function generateItemCode() {
  // 从localStorage读取计数器
  let counter = parseInt(localStorage.getItem('itemCodeCounter') || '1');
  
  // 生成编码
  const code = `SKU${String(counter).padStart(5, '0')}`;
  
  // 递增并保存
  counter++;
  localStorage.setItem('itemCodeCounter', counter.toString());
  
  return code;
}
```

**特点：**
- 🔒 跨会话保持唯一
- 📊 格式化输出（前导零）
- 💾 本地持久化
- ⚡ 高效生成

## 🐛 已知限制

### 当前版本的限制

1. **数据存储**
   - 使用localStorage，清除缓存会丢失数据
   - 单设备存储，不支持多设备同步
   - 存储空间有限（通常5-10MB）

2. **历史记录**
   - 暂不支持手动清除历史
   - 历史记录永久保存，可能累积过多
   - 无法导出/导入历史数据

3. **类别管理**
   - 暂不支持类别编辑和删除
   - 类别层级仅支持一级
   - 无法合并重复类别

4. **商品编码**
   - 不支持自定义编码规则
   - 编码不可修改或重置
   - 无编码校验机制

### 解决方案（未来版本）

- v1.4：集成Supabase，解决数据持久化问题
- v1.5：增加历史记录管理功能
- v2.0：支持多级分类和类别编辑
- v2.1：支持自定义编码规则
